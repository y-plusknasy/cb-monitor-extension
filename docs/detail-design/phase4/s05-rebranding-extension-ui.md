# S05: リブランディング + Extension UI 刷新 — 詳細設計書

## 1. 概要

フェーズ 4 (UX 改善・配布準備) の第 1 スプリントとして、アプリ名統一 (「CB Link」) および Extension の popup / options ページの UI 刷新を行う。

**スコープ:**

- アプリ名「Web Usage Tracker」→「CB Link」への統一（manifest.json、UI テキスト、ログ出力）
- Popup UI 刷新（モダンなデザイン、アプリ別利用時間内訳表示）
- Options ページ UI 刷新（セクション整理、ペアリング済みデバイス名編集、状態表示改善）
- Service Worker のログプレフィックス更新

**スコープ外:**

- 新アイコンの作成（S05 では既存アイコンを流用。別途デザイン確定後に差し替え）
- APK ビルド・Chrome Store 公開（S06）
- モバイルアプリ UI 改善（S07）

---

## 2. 機能要件・受け入れ基準

### 2.1 リブランディング

**要件**: アプリ名を「CB Link」に統一する。

| 対象            | 変更内容                                               |
| --------------- | ------------------------------------------------------ |
| `manifest.json` | `name`: "Web Usage Tracker" → "CB Link"                |
| `manifest.json` | `description`: 更新                                    |
| `package.json`  | `name`: "cb-monitor-extension" → "cb-link-extension"   |
| `package.json`  | `description`: 更新                                    |
| Popup HTML      | タイトル "Web Usage Tracker" → "CB Link"               |
| Options HTML    | タイトル "Web Usage Tracker — 設定" → "CB Link — 設定" |
| Service Worker  | ログプレフィックス `[WebUsageTracker]` → `[CBLink]`    |
| Options JS      | ログプレフィックス `[WebUsageTracker]` → `[CBLink]`    |

**受け入れ基準:**

- すべての UI テキスト・ログ出力から "Web Usage Tracker" の文字列が消えていること
- 拡張機能名として Chrome ブラウザ上に "CB Link" と表示されること

### 2.2 Popup UI 刷新

**要件**: ステータス情報をよりわかりやすく表示し、アプリ別の利用時間内訳を追加する。

**現行 Popup:**

- 計測対象、登録状態、デバイスID、本日の合計 の 4 行表示

**新 Popup:**

- ヘッダー: "CB Link" ロゴテキスト + ペアリング状態インジケーター（ドット）
- ステータスセクション: 計測中アプリ名（アクティブ/非アクティブ表示）
- 本日の合計: 大きなフォントで表示
- アプリ別内訳: 上位アプリの利用時間バー（dailyUsage から取得）
- フッター: デバイスID（短縮表示）+ 設定リンク

**受け入れ基準:**

- Popup に今日のアプリ別利用時間内訳が表示される
- ペアリング済み/未登録が視覚的に区別できる（カラーインジケーター）
- スクロールなしで主要情報が確認できる（幅 320px、高さ 400px 以内）
- 設定ページへのリンクが Popup 内に存在する

### 2.3 Options ページ UI 刷新

**要件**: 設定項目を整理し、ペアリング状態の視認性を高め、デバイス名の編集機能を追加する。

**現行 Options:**

- API 接続設定セクション（エンドポイント URL 入力）
- デバイス登録セクション（OTP + デバイス名入力）

**新 Options:**

- ヘッダー: "CB Link — 設定"
- ペアリング状態カード: 登録済みの場合はデバイス名・登録日・デバイスID を表示。未登録の場合はセットアップ案内。
- デバイス名編集: ペアリング済みの場合、インラインでデバイス名を編集可能（chrome.storage.local + sync バックアップを更新）
- OTP 登録フォーム: 未登録時のみ表示（デバイス名 + OTP 入力）
- API エンドポイント設定: 折りたたみ可能な詳細設定セクション

**受け入れ基準:**

- ペアリング済み時にデバイス名が編集できる
- デバイス名の変更が chrome.storage.local と chrome.storage.sync に反映される
- 未登録時は OTP 入力フォームが表示される
- API エンドポイントが折りたたみセクション内で設定可能

### 3.4 デバイス名編集機能

**動作仕様:**

1. ペアリング済みの場合、デバイス名の横に編集ボタン（✎）を表示
2. 編集ボタンクリック → デバイス名が input フィールドに変わる
3. Enter キーまたは保存ボタンで確定:
   - `chrome.storage.local` の `pairingStatus.deviceName` を更新
   - `chrome.storage.sync` のバックアップも更新（利用可能な場合）
4. Escape キーでキャンセル

**注意**: デバイス名の変更は Extension ローカルのみ。Firestore の `devices` / `users` コレクションへの反映は行わない（サーバー側の名前更新は将来のモバイルアプリ機能で対応予定）。
