# ADR-012: タブスワイプ切り替え機能の見送り

## ステータス

却下（Rejected）

## 背景

モバイルアプリの「ホーム」「デバイス」「設定」タブを、ボトムタブバーのボタンタップに加えて、画面の水平スワイプでも切り替えたいという要望があった。

## 検討した選択肢

### 選択肢 A: `react-native-gesture-handler` でスワイプ検知 → `router.navigate()`

- 各タブ画面をジェスチャーハンドラーでラップし、水平スワイプで遷移
- **メリット**: 既存の Expo Router 構造を壊さない、軽量実装
- **デメリット**: 指に追従するスライドアニメーションはなく、スワイプ完了後にタブが切り替わる

### 選択肢 B: `react-native-pager-view` でコンテンツをラップ

- タブレイアウト内で PagerView にコンテンツを載せ、スワイプ状態をタブと双方向同期
- **メリット**: ネイティブのスワイプアニメーション（指追従）
- **デメリット**: Expo Router のタブ管理と衝突しやすく、同期ロジックが複雑

### 選択肢 C: `@react-navigation/material-top-tabs` に置き換え

- スワイプ対応のタブナビゲーターに丸ごと置き換え
- **メリット**: スワイプがネイティブ対応
- **デメリット**: Expo Router のファイルベースルーティングとの統合が困難。ボトムタブ UI を失う

## 決定

**v1.0 では実装しない。** 今後のユーザーフィードバック次第で選択肢 A での導入を検討する。

## 理由

- 現在のボトムタブ切り替えで基本的な操作性は満たしている
- 選択肢 A が最も低リスクだが、v1.0 リリースの優先度には含まれない
- リリース後のフィードバックで要望が多ければ導入を再検討する
